# KG Nexus 知识图谱问答系统 · 同学操作指南（新版）

> 适用对象：拿到项目后，需要在 Windows 上完成环境配置、启动 Web 页面、导入 Neo4j、使用问答和图谱控制台的同学。  
> 项目主题：**中国先进人工智能技术知识图谱问答系统**  
> 当前数据规模：**377 个实体，954 条关系，共 1331 条图谱记录**。

---

## 一、项目能做什么

本项目是一个面向“中国先进人工智能技术”领域的知识图谱问答系统，主要功能包括：

1. **知识图谱可视化**：浏览实体和关系，支持搜索、聚焦、关系网查看、全屏查看。
2. **智能问答**：支持 KG-only 和 KG + LLM 两种问答模式。
3. **问答联动图谱**：问答结果可同步高亮图谱中的相关实体。
4. **分屏对比**：问答页可一键左右分屏，左侧问答，右侧图谱。
5. **实体数据表格**：查看、筛选所有实体信息。
6. **图谱控制台**：输入 Neo4j Cypher 语句进行查询、新增、修改、删除。
7. **结果导出**：控制台查询结果可导出为 CSV 或 JSON。
8. **评测报告**：可运行脚本生成 KG-only 与 KG+LLM 的问答评测报告。

## 四、配置 `.env`

如果没有 `.env` 文件，先复制模板：

```powershell
copy .env.example .env
```

`.env` 主要配置如下：

```env
# 图谱模式：neo4j 或 local
GRAPH_MODE=neo4j

NEO4J_URI=bolt://localhost:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=kgqa123456

# 大模型，可选
LLM_ENABLED=false
LLM_API_KEY=
LLM_BASE_URL=https://api.deepseek.com/v1
LLM_MODEL=deepseek-v4-flash

API_HOST=0.0.0.0
API_PORT=8001
```

说明：

- `GRAPH_MODE=local`：使用本地 JSON 图谱，不需要 Neo4j。
- `GRAPH_MODE=neo4j`：使用 Neo4j 图数据库，答辩时更推荐。
- `NEO4J_PASSWORD` 必须和 Neo4j Desktop 中数据库密码一致。
- `.env` 里如果写了 API Key，不要打包发给别人。

## 五、两种图谱模式

### 5.1 本地 JSON 模式

适合快速演示，不需要安装 Neo4j。

修改 `.env`：

```env
GRAPH_MODE=local
```

然后直接启动：

```powershell
conda activate kgqa
cd C:\Users\WRQ\Desktop\knowledgegraph
python run_server.py
```

浏览器打开：

```text
http://localhost:8001
```

本地模式可以使用：

- 知识图谱可视化
- 智能问答
- 实体表格
- 分屏对比
- 评测脚本

但 **本地模式不支持 Cypher 控制台执行 SQL/Cypher**，因为 JSON 文件不是图数据库。

### 5.2 Neo4j 模式

适合正式答辩，支持图数据库和 Cypher 控制台。

步骤：

1. 打开 Neo4j Desktop。
2. 创建 Local DB。
3. 设置密码，例如 `kgqa123456`。
4. 点击 Start 启动数据库。
5. `.env` 中设置：

```env
GRAPH_MODE=neo4j
NEO4J_URI=bolt://localhost:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=你的真实密码
```

6. 导入数据：

```powershell
python scripts/01_import_data.py
```

成功时终端会显示节点数和关系数。

## 七、页面功能说明

### 7.1 知识图谱

功能包括：

- 加载全图
- 设置渲染节点数量
- 搜索实体
- 聚焦实体
- 单击节点进入实体关系网
- 全屏查看
- 图例高亮：当前实体、问句关键词、答案关联实体

“渲染节点数量”可以选择：

```text
80 / 120 / 160 / 200 / 260
```

切换后会重新请求图谱并重新渲染。

### 7.2 智能问答

支持两种模式：

```text
KG-only
KG + LLM
```

KG-only：只基于知识图谱回答，速度快，可解释性强。  
KG + LLM：先从图谱检索证据，再调用大模型生成更自然的回答。

推荐测试问题：

```text
文心一言是什么？
百度研发了哪些AI产品？
深度学习属于什么技术领域？
DeepSeek是哪个国家的产品？
自动驾驶应用了哪些技术？
```

### 7.3 分屏对比

在智能问答页点击：

```text
分屏对比
```

页面会分成两部分：

- 左侧：智能问答
- 右侧：知识图谱

适合答辩时展示“问答结果如何对应图谱实体”。

再次点击：

```text
退出分屏
```

即可恢复普通问答页面。

### 7.4 实体数据

实体数据页展示所有实体，包括：

- 名称
- 类型
- 描述
- 年份
- 地区
- 参考文献

可以使用上方输入框筛选实体名称、类型、描述或参考文献。

### 7.5 图谱控制台

图谱控制台用于执行 Neo4j Cypher 语句。

注意：

```text
只有 Neo4j 模式支持图谱控制台。
本地 JSON 模式不支持 Cypher。
```

常用查询：

```cypher
MATCH (n:Entity)
RETURN n.name AS name, n.type AS type
LIMIT 10
```

查询某个实体的出边：

```cypher
MATCH (n:Entity {name:'百度'})-[r:RELATION]->(m:Entity)
RETURN n.name AS source, r.type AS relation, m.name AS target
```

新增实体：

```cypher
CREATE (:Entity {
  id:'E999',
  name:'测试实体',
  type:'Technology',
  description:'示例',
  year:2026,
  country:'中国',
  reference:'手动添加'
})
```

新增关系：

```cypher
MATCH (a:Entity {name:'人工智能'}), (b:Entity {name:'测试实体'})
CREATE (b)-[:RELATION {
  type:'属于',
  description:'测试实体属于人工智能',
  reference:'手动添加'
}]->(a)
```

删除实体：

```cypher
MATCH (n:Entity {name:'测试实体'})
DETACH DELETE n
```

图谱控制台功能：

- `?` 按钮：查看示例弹窗
- `执行`：执行 Cypher
- `Ctrl + Enter`：快捷执行
- `渲染表格`：把查询结果显示成平铺表格
- `原始 JSON`：显示原始 JSON
- `导出结果`：导出 CSV 或 JSON

导出规则：

- 表格模式导出 `.csv`
- 原始 JSON 模式导出 `.json`

## 十、常见问题

### 1. Neo4j 认证失败

报错类似：

```text
Neo.ClientError.Security.Unauthorized
authentication failure
```

原因：

```text
.env 里的 NEO4J_PASSWORD 和 Neo4j Desktop 真实密码不一致
```

解决：

1. 打开 Neo4j Desktop。
2. 确认当前数据库密码。
3. 修改 `.env`：

```env
NEO4J_PASSWORD=你的真实密码
```

4. 重启服务或重新导入。

如果忘记密码，最简单的方法是新建一个 Local DB，密码设为 `kgqa123456`，并同步修改 `.env`。

### 2. 图谱控制台执行失败：Not Found

原因：

```text
后端服务没有重启，当前运行的还是旧代码
```

解决：

```powershell
Ctrl + C
python run_server.py
```

然后浏览器 `Ctrl + F5`。

### 3. 图谱控制台提示本地模式不支持 Cypher

原因：

```text
当前 GRAPH_MODE=local
```

解决：

修改 `.env`：

```env
GRAPH_MODE=neo4j
```

启动 Neo4j Desktop 数据库，并导入数据：

```powershell
python scripts/01_import_data.py
```

### 4. 端口 8001 被占用

查看端口：

```powershell
netstat -ano | findstr :8001
```

结束占用进程：

```powershell
taskkill /PID 进程号 /F
```

或者修改 `.env`：

```env
API_PORT=8002
```

浏览器改访问：

```text
http://localhost:8002
```

### 5. 页面看不到新功能

原因：

```text
浏览器缓存了旧版 JS/CSS
```

解决：

```text
Ctrl + F5 强制刷新
```

如果仍然不行，重启后端：

```powershell
python run_server.py
```

### 6. 图谱加载后节点一直乱动

当前版本已经对图谱布局做了优化：

- 多核心布局
- 连通主图采样
- 保留动态展开效果
- 支持渲染数量设置

如果节点数量较多仍然卡顿，可以在知识图谱页把渲染数量调低，例如：

```text
120 节点 或 160 节点
```
