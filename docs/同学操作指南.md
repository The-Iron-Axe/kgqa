## 一、你需要准备什么

| 软件 | 版本建议 | 用途 | 下载地址 |
|------|----------|------|----------|
| **Python** | 3.10 或 3.11（64 位） | 运行后端 | https://www.python.org/downloads/ |
| **Neo4j Desktop** | 最新版 | 图数据库（答辩推荐） | https://neo4j.com/download/ |
| 浏览器 | Chrome / Edge | 访问演示页面 | 系统自带即可 |

**说明：**

- **不装 Docker 也可以**，本项目推荐用 **Neo4j Desktop**（图形界面，适合课程设计）。
- 若暂时装不了 Neo4j，可先用 **本地模式**（见第十节），无需数据库即可演示界面和问答。

## 二、解压与目录结构

1. 将收到的压缩包解压到**路径中尽量没有中文和空格**的位置，例如：  
   `D:\knowledgegraph`
2. 解压后应能看到类似结构：

```
knowledgegraph/
├── run_server.py          ← 主程序：启动 Web 服务
├── requirements.txt       ← Python 依赖列表
├── .env.example           ← 环境变量模板（需复制为 .env）
├── config/                ← 配置文件
├── data/
│   └── raw/
│       └── ai_knowledge.json   ← 知识图谱原始数据
├── scripts/
│   ├── 01_import_data.py       ← 导入 Neo4j（Neo4j 模式必跑）
│   └── 02_run_evaluation.py    ← 生成评测报告（可选）
├── src/                   ← 后端源码
├── frontend/              ← 前端页面
└── docs/                  ← 说明文档（含本指南）
```

## 三、安装 Python 与项目依赖

### 3.1 安装 Python

1. 打开 https://www.python.org/downloads/ 下载 Windows 安装包。
2. 安装时**务必勾选**「Add Python to PATH」。
3. 安装完成后打开 **PowerShell** 或 **命令提示符**，输入验证：

```powershell
python --version
```

应显示 `Python 3.10.x` 或 `3.11.x`。

### 3.2 进入项目目录

```powershell
cd D:\knowledgegraph
```

（把路径改成你实际解压的位置。）

### 3.3 创建虚拟环境（推荐）

```powershell
python -m venv .venv
.\.venv\Scripts\activate
```

若 PowerShell 提示「禁止运行脚本」，先执行（只需一次）：

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

然后再执行 `activate`。

**不激活虚拟环境也可以**：后续命令把 `python` 换成：

```powershell
D:\knowledgegraph\.venv\Scripts\python.exe
```

### 3.4 安装依赖

```powershell
pip install -r requirements.txt
```

国内网络慢可使用清华源：

```powershell
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

## 四、下载并配置 Neo4j（推荐答辩使用）

### 4.1 下载 Neo4j Desktop

1. 打开：https://neo4j.com/download/
2. 选择 **Neo4j Desktop**（不是 Server 单独安装包）。
3. 注册免费账号后下载 Windows 安装程序并完成安装。

### 4.2 创建本地数据库

1. 打开 **Neo4j Desktop**。
2. 点击 **New** → **Create project**（项目名称随意，如 `KGQA`）。
3. 在项目里点击 **Add** → **Local DB**：
   - **Name**：随意，如 `knowledgegraph`
   - **Password**：自己设置并**记住**（下文示例用 `kgqa123456`，请与 `.env` 中一致）
   - **Version**：选 5.x（如 5.26）即可
4. 创建完成后，点击该数据库右侧的 **Start**，状态变为 **Running**（绿色）。

### 4.3 确认连接信息

默认连接参数一般为：

| 项 | 值 |
|----|-----|
| 连接地址 | `bolt://localhost:7687` |
| 用户名 | `neo4j` |
| 密码 | 你在 Desktop 里设置的密码 |

可在数据库卡片上点 **Open** → **Open with Browser**，用浏览器插件或 Neo4j Browser 查看（可选，用于检查是否启动成功）。

### 4.4 防火墙提示

若 Windows 防火墙询问是否允许 Neo4j 访问网络，选择**允许**（专用网络即可）。

## 五、配置项目环境变量 `.env`

1. 在项目根目录找到 `.env.example`。
2. **复制一份**并重命名为 `.env`（与 `run_server.py` 同级）。
3. 用记事本或 PyCharm 打开 `.env`，至少确认以下配置：

```env
# 使用 Neo4j 图数据库（答辩推荐）
GRAPH_MODE=neo4j

NEO4J_URI=bolt://localhost:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=kgqa123456

# Web 服务端口（浏览器访问用 localhost:8001）
API_HOST=0.0.0.0
API_PORT=8001

# 大模型默认关闭；需要时再改（见第九节）
LLM_ENABLED=false
```

**重要：**

- 把 `NEO4J_PASSWORD` 改成你在 Neo4j Desktop 里设置的**真实密码**。
- 不要把含真实 API Key 的 `.env` 再打包发给别人。

## 六、导入知识图谱数据到 Neo4j

**前提：** Neo4j Desktop 里数据库已是 **Running** 状态。

在项目根目录执行（已激活虚拟环境时）：

```powershell
python scripts/01_import_data.py
```

或在 PyCharm 中：右键 `scripts/01_import_data.py` → **Run**。

**成功标志：** 终端输出节点数、关系数，无报错。脚本会把 `data/raw/ai_knowledge.json` 写入 Neo4j。

**若提示当前为 local 模式、无需导入：** 说明 `.env` 里 `GRAPH_MODE` 不是 `neo4j`，请检查第五节。

**若连接失败（localhost:7687 拒绝连接）：**

- 确认 Neo4j Desktop 中数据库已 **Start**。
- 确认 `.env` 中密码正确。

## 七、启动 Web 服务并访问页面

### 7.1 启动服务

```powershell
python run_server.py
```

或 PyCharm 右键 `run_server.py` → **Run**。

**成功标志：** 终端出现类似：

```
==================================================
  服务启动后，请在浏览器打开：
  http://localhost:8001
  图谱模式配置: neo4j（Neo4j 未启动时将自动回退 local）
==================================================
INFO:     Uvicorn running on http://0.0.0.0:8001
```

### 7.2 打开网页

浏览器访问（不要用 `0.0.0.0`）：

**http://localhost:8001**

### 7.3 停止服务

在运行 `run_server.py` 的终端按 **`Ctrl + C`**，等出现 `Shutting down` 后再关窗口。  
否则端口会被占用，下次启动报「地址只允许使用一次」。

## 八、页面功能说明

| 标签页 | 功能 |
|--------|------|
| **知识图谱** | 全图浏览、搜索实体、点击进入关系网；可全屏 |
| **智能问答** | 输入问题，选择 **KG-only** 或 **KG + LLM**；回答后可点「查看图谱中的高亮」 |
| **实体数据** | 表格浏览全部实体；问答后匹配关键词会置顶高亮 |
| **评测对比** | 说明如何生成 KG vs KG+LLM 评测报告 |

**推荐试玩问题（图谱里都有对应实体）：**

- `文心一言是什么？`
- `百度研发了哪些AI产品？`
- `深度学习属于什么技术领域？`

## 九、可选：启用大模型（KG + LLM）

1. 注册 DeepSeek 等 OpenAI 兼容 API：https://platform.deepseek.com/
2. 在 `.env` 中修改：

```env
LLM_ENABLED=true
LLM_API_KEY=你的密钥
LLM_BASE_URL=https://api.deepseek.com/v1
LLM_MODEL=deepseek-chat
```

3. 重启 `run_server.py`，在网页问答模式选 **KG + LLM**。

未配置 API Key 时，选 KG + LLM 会自动回退为纯图谱回答。

## 十、可选：不用 Neo4j 的本地模式

若电脑装 Neo4j 有困难，可快速演示：

1. 编辑 `.env`：

```env
GRAPH_MODE=local
```

2. **不要**运行 `01_import_data.py`。
3. 直接运行 `run_server.py`，浏览器打开 http://localhost:8001。

数据来自 `data/raw/ai_knowledge.json`，功能与界面基本一致。答辩时可说明：「开发阶段用 JSON，正式环境可切换 Neo4j」。

> 若 `GRAPH_MODE=neo4j` 但 Neo4j 未启动，系统也会**自动回退**到本地 JSON，并在终端提示。

## 十二、常见问题排查

### Q1：端口 8001 被占用

```powershell
netstat -ano | findstr :8001
taskkill /PID <最后一列PID> /F
```

或修改 `.env` 中 `API_PORT=8002`，浏览器改访问对应端口。

### Q2：Neo4j 连接失败

- Desktop 数据库是否 **Running**？
- `.env` 密码是否与 Desktop 一致？
- 是否先运行 `01_import_data.py`？

### Q3：页面能开但图谱空白

- 按 **Ctrl + F5** 强制刷新。
- 点「加载全图」。
- 确认 `01_import_data.py` 已成功执行。

### Q4：连续问答后图谱变卡

- 已优化为：问答时不反复重载全图，点「查看图谱高亮」时才加载。
- 请用压缩包内**最新代码**，并强制刷新浏览器缓存。

### Q5：`pip install` 很慢或失败

使用国内镜像（见 3.4 节），或检查 Python 是否为 64 位 3.10+。

### Q6：PyCharm 提示找不到模块

**Run → Edit Configurations** → 选中 `run_server`：

- **Working directory**：项目根目录（含 `run_server.py` 的文件夹）
- 解释器选项目的 `.venv`

## 十三、答辩演示建议顺序

1. **启动 Neo4j Desktop**（显示 Running）。
2. **运行** `run_server.py`，打开 http://localhost:8001 。
3. **知识图谱**：展示全图 → 搜索「文心一言」→ 进入关系网。
4. **智能问答**：同一实体问 1～2 个问题 → 点「查看图谱中的高亮」。
5. **实体数据**：展示问答后关键词置顶高亮。
6. **评测对比**：打开已生成的 `evaluation_report.md`（若已跑评测脚本）。
7. 口头说明：领域为「中国先进 AI」、图谱存于 Neo4j、问答为「图谱检索 + 模板/可选 LLM」。
